## 2024-05-24 - Missing Local Storage Input Validation
**Vulnerability:** The application was trusting data retrieved from `localStorage` (`localStorage.getItem("spaceInvadersHighScore")`) directly without any validation, simply parsing it with `parseInt()` without verifying if the result was `NaN` or a valid non-negative integer.
**Learning:** This is a form of DOM-based injection/state corruption. Even though `localStorage` is isolated per origin, a compromised script or an external factor could inject invalid data (like negative numbers, strings, or extremely large numbers). If the application uses this tampered data without validation, it can lead to state corruption (e.g., negative high score, `NaN` displaying in UI) and potentially unexpected logic flaws in the game if this state is used for calculations. Trusting `localStorage` blindly is a security anti-pattern because the client fully controls it.
**Prevention:** Always validate and sanitize data retrieved from client-side storage mechanisms (`localStorage`, `sessionStorage`, cookies, IndexedDB) before using it in application logic. Ensure it conforms to expected types, ranges, and formats. In this case, checking for `!isNaN` and `value >= 0` is necessary for a high score. If invalid data is found, it should be discarded or reset to a safe default.
